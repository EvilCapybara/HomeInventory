# Code Review — `main.py`
**Date:** 2026-06-02  
**Reviewer:** code-quality-reviewer agent

---

## Critical Issues

### C1 — Bot initialized at module level (lines 10–11)
`Bot(token=...)` and `register_handlers()` execute on every import, not just when running directly. Any test importing `main` will immediately attempt a Telegram API connection.

### C2 — `config.API_TOKEN` not validated before use
If `TELEGRAM_TOKEN` is unset in `.env`, `None` is silently passed to `Bot(token=None)`. The error surfaces deep inside pyTeleBot with no useful context.

---

## Major Issues

### M1 — `except Exception as e: raise` is a no-op
Catches the exception and immediately re-raises it. Variable `e` is never used. No logging, no handling.

### M2 — `logging` imported but never configured
`basicConfig()` is never called → all `logger.info(...)` calls from `homemanager.py` and other modules don't output properly.

### M3 — `import cli` is a dead import
`cli.py` is unused in `main.py`; the only references to it are commented out.

---

## Minor Issues

- `print("Bot is waiting...")` should be `logger.info(...)`
- Commented-out dead code in `main()`
- `home_manager` variable assigned only for singleton side effect — implicit and confusing

---

## Suggested Fix

```python
import logging
import config

logger = logging.getLogger(__name__)


def main() -> None:
    logging.basicConfig(
        level=logging.INFO,
        format="%(asctime)s %(levelname)s %(name)s %(message)s",
    )

    if not config.API_TOKEN:
        raise EnvironmentError("TELEGRAM_TOKEN is not set.")

    from api import Bot
    from homemanager import HomeManager

    HomeManager()  # initialize singleton: DB + ES index

    bot = Bot(token=config.API_TOKEN)
    bot.register_handlers()

    logger.info("Bot is running.")
    bot.infinity_polling()


if __name__ == '__main__':
    try:
        main()
    except Exception:
        logger.exception("Fatal error during bot startup")
        raise SystemExit(1)
```
